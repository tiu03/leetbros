### #1 Brute force method
This approach asks the following questions:
1. How to turn array into pairs?
   -> Use nested loop
```python
class Solution(object):
    def twoSum(self, nums, target):
	    for i in range(len(nums)):
		    for j in range(i+1, len(nums)):
			    if nums[i] + nums[j] == target:
				    return [i,j]
				

```

### #2 Optimal method: 

### a) Two-pass
This approach asks the following questions:
1. Could perfect information help?
   -> Try using data structure to store the array
2. With perfect information, how to check if the number we need at index i exists in the array?
   -> Calculate target - nums\[i]
3. What data structure is suitable?
   -> Python has List, Dictionary, Tuple, Set

```python
myDict = {}

for i in range(len(nums)):
	myDict[nums[i]] = i

for i in range(len(nums)):
	ideal_j = target - nums[i]
	if ideal_j in myDict and myDict[ideal_j] != i:
		return [i, myDict[ideal_j]]
```

### b) One-pass
This approach asks the following questions:
1. Could we eliminate the first for loop in the two pass method?
   -> Yes, by storing elements in hashmap during the second loop.
   At loop start, x and y are some distance apart. When y is reached, x (ideal j for y) already exists in the map and hence the solution is found.
2. Would it improve the time & space complexity
   -> Yes, but not significantly
3. Is this performance optimisation favourable over design clarity?
   -> Depends on the use case, for explaining purposes it is preferred to teach 2 pass before 1 pass.

A hash (a.k.a hash table, hash map) is a data structure that uses a hash function to store key-value pairs. The hash function creates a hash (key) that maps to a specific value in the hash table. 

For a key value pair $x:v$ 
$f : x \rightarrow n$

where $n$ represents the $n^{th}$ slot of the hash table where the value, $v$ is stored. The hash function must return the same value for the same input (1-to-1).

Python dictionaries are implemented as hash tables, and can handle collisions using techniques like chaining or open addressing.

Like every data structure, hashmaps are buillt on top of arrays. So the questions is, how can keys become a unique index of an array that we can use for direct lookup?
